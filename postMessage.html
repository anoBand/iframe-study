양방향 통신 (postMessage)

부모 창과 자식 iframe 간의 실시간 메시징 데모

부모 -> 자식 수신함
메시지를 기다리는 중...
자식 -> 부모 전송

보안 가이드라인

  • event.origin을 반드시 확인하여 신뢰할 수 있는 도메인인지 검증해야 합니다.
  • targetOrigin을 '*' 대신 명확한 URL로 지정하는 것이 안전합니다.

실시간 양방향 통신

부모 창과 자식 iframe이 물리적으로 분리되어 있어도 메모리 상에서 직접 데이터를 주고받아 즉각적인 UI 반응을 구현합니다.

상태 기반 트리거

'상태 전송'과 같은 특정 이벤트를 부모에게 보고하여, 부모 창의 알림(Toast)이나 워크플로우를 제어하는 트리거로 활용합니다.
